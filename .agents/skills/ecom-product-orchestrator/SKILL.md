## 输入参数

```json
{
  "site": "amazon.com",
  "category": "home-garden",
  "product_count": 5,
  "price_range": {
    "min": 20,
    "max": 100
  },
  "exclude_categories": ["医疗", "婴幼儿食品"],
  "strategy_preference": ["bundle", "premium", "lightweight"],
  "output_format": "full"
}
```

| 参数 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `site` | string | 否 | amazon.com | 亚马逊站点 |
| `category` | string | 否 | home-garden | 商品类目 |
| `product_count` | int | 否 | 5 | 返回产品数量 |
| `price_range` | object | 否 | 无 | 价格区间（美元） |
| `exclude_categories` | list | 否 | [] | 排除类目 |
| `strategy_preference` | list | 否 | 全部 | 优先策略 |
| `output_format` | string | 否 | full | 输出格式 |

## 核心筛选逻辑

### Step 1: 市场源头筛选

从亚马逊飙升榜获取候选产品后，执行以下过滤：

1. **排除高资质门槛类**
   - 医疗器械 → 跳过
   - 婴幼儿食品 → 跳过
   - 药品保健品 → 跳过

2. **排除品牌垄断类**
   - Apple 配件 → 跳过
   - Samsung 配件 → 跳过

3. **排除价格过低类**
   - 售价 < $10 → 跳过（利润空间不足）

### Step 2: 竞争格局筛选

根据 Temu 竞品分析：

1. **竞争过于激烈**（竞品 > 100，价格极低）
   - 标记为 "红海"
   - 需强差异化策略才能进入

2. **中度竞争**（竞品 20-50）
   - 标记为 "黄海"
   - 可通过差异化进入

3. **蓝海机会**（竞品 < 20）
   - 标记为 "蓝海"
   - 优先推荐

### Step 3: 利润红线筛选

严格执行 V4.1 核价模型：

```
净利润 = 回款预估(人民币) - 总成本
净利润 > ¥5.00 → GO
净利润 ≤ ¥5.00 → PASS
```

### Step 4: 综合评分排序

对通过筛选的产品按以下维度评分：

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 利润率 | 40% | 净利润/售价 |
| 市场热度 | 25% | 亚马逊排名变化 |
| 竞争程度 | 20% | Temu 竞品数量 |
| 供应链稳定性 | 15% | 工厂数量、产能 |

## 执行流程

### 自动执行模式

当用户发起选品请求时，自动执行以下步骤：

```
1. 调用 amazon-movers-shakers
   - 获取飙升榜候选池（默认 100 个）
   - 执行初步过滤

2. 对每个候选产品：
   a. 调用 temu-competitor-search
      - 获取卷王价
      - 分析竞争格局
   
   b. 调用 ali1688-sourcing
      - 获取起批价
      - 确认供应链
   
   c. 调用 temu-pricing-calculator
      - 计算净利润
      - 判定 GO/PASS

3. 过滤 PASS 产品
4. 按综合评分排序
5. 返回 Top 5 推荐产品
```

### 交互式执行模式

当用户需要逐步确认时：

```
Step 1: 展示亚马逊飙升榜候选池
        → 用户确认/调整候选列表

Step 2: 展示 Temu 竞品分析
        → 用户确认竞争策略

Step 3: 展示 1688 供应链
        → 用户确认工厂选择

Step 4: 展示核价结果
        → 用户确认最终推荐
```

## 与其他 Skills 协作

| Skill | 协作方式 |
|-------|----------|
| `amazon-movers-shakers` | 调用获取候选产品池 |
| `temu-competitor-search` | 调用获取卷王价和竞品分析 |
| `ali1688-sourcing` | 调用获取拿货价和供应链 |
| `temu-pricing-calculator` | 调用进行核价计算 |

## 注意事项

1. **数据时效性**：飙升榜数据每小时更新，建议定期刷新

2. **价格波动**：Temu 和 1688 价格变动频繁，建议多次采样

3. **供应链验证**：1688 价格需实际询价确认

4. **认证要求**：出口产品需确认认证证书

5. **样品确认**：建议先拿样品确认品质
