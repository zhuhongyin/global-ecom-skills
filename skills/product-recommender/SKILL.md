# Product Recommender

## 概述

此Skill是选品流程的整合模块，将亚马逊趋势分析、Temu竞品分析、1688货源查找和利润计算的结果整合，生成最终的选品推荐报告。

## 使用场景

当用户需要：
- 获取完整的选品推荐
- 整合多个分析结果
- 生成可执行的选品报告
- 为采购提供决策依据

## 核心筛选逻辑

### 三大避卷策略

1. **组合拳策略（Bundle）**
   - 卖套装、卖多件装
   - 如：50件智能家居套餐
   - 包括桌板、桌腿、桌椅、支架等配套商品

2. **视觉欺诈（Premium）**
   - 卖材质/设计看起来更贵的差异化款
   - 提升产品质感

3. **极致轻小（Lightweight）**
   - 运费极低的小件
   - 如：桌垫、配件等

### 利润红线（V4.1生存模型）

```
回款预估 = Temu前台卷王价 × 45%
总成本 = 1688起批价 + ¥3.50
净利润 = (回款预估 × 7.2) - 总成本

【铁律】净利润必须 > ¥5.00
```

## 执行步骤

### 步骤1：收集分析数据

整合以下模块的输出：

| 模块 | 数据来源 | 关键信息 |
|------|---------|---------|
| amazon-trend-analyzer | 亚马逊飙升榜 | 热销产品、趋势 |
| temu-competitor-analyzer | Temu竞品 | 价格、竞争程度 |
| supply-platform-analyzer | 4Supply平台 | B2B价格参考 |
| sourcing-1688 | 1688货源 | 采购价格、供应商 |
| profit-calculator | 利润计算 | 净利润判定 |

### 步骤2：产品筛选

按以下标准筛选产品：

| 筛选维度 | 通过标准 |
|---------|---------|
| 亚马逊热度 | 飙升榜前100或飙升幅度>100% |
| Temu竞争 | 搜索结果<5000，有差异化空间 |
| 利润 | 净利润>¥5.00 |
| 合规 | 无高门槛认证要求 |
| 物流 | 轻小件优先 |

### 步骤4：输出执行建议

为每个推荐产品提供：

1. **采购建议**
   - 推荐供应商
   - 建议采购量
   - 质量检验要点

2. **上架建议**
   - 建议定价
   - 主图设计要点
   - 标题关键词

3. **运营建议**
   - 初期推广策略
   - 库存管理建议
   - 售后注意事项

## 质量控制

### 宁缺毋滥原则

- 如果计算利润不足，明确标注不推荐
- 不推荐高合规门槛产品
- 不推荐物流成本过高的产品

### 推荐产品数量

- 每次推荐3-5款产品
- 确保每款都经过完整分析
- 提供充分的推荐理由

## 注意事项

1. **数据时效性**
   - 使用最新的市场数据
   - 关注季节性因素

2. **合规审查**
   - 避免侵权产品
   - 注意认证要求

3. **风险提示**
   - 市场风险
   - 供应链风险
   - 平台政策风险
