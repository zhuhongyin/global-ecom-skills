## Temu/4supply 选品智能体需求拆解

基于 `requirement.md` 与实施计划中的 N1–N13，将“大选品智能体”拆解为以下独立可实现的小需求（每个对应一个 Claude skill 或 orchestrator 流程节点）。

### 一、数据获取类（N1–N4）

- **N1：亚马逊飙升榜数据抓取/查询**
  - 目标：从 Amazon Movers & Shakers 获取近期飙升商品池，为后续筛选提供“市场源头”数据。
  - 输入：
    - 站点（如 `amazon.com`, `amazon.de`）
    - 类目或关键词
    - 时间窗口（如最近 24 小时 / 7 天）
  - 输出：
    - 商品基础信息：标题、ASIN、链接、缩略图
    - 价格、主图价区间
    - 排名与近期排名/销量趋势（如是否快速上升）

- **N2：Temu 竞品与卷王价查询**
  - 目标：在 Temu 上找到对应竞品，并给出最低“卷王价”和核心卖点摘要。
  - 输入：
    - 关键词/品名（可由 N1 标题加工）
    - 类目或粗粒度品类标签
  - 输出：
    - 同类产品列表（SKU 名称、链接、销量/评分信号）
    - 最低价（卷王价）
    - 主要卖点与差异点摘要（如套装、多件装、品牌感设计等）

- **N3：4supply 商品与机会位洞察**
  - 目标：判断 4supply 上是否已有对应商品，价格区间如何，是否已“卖爆”或仍有空间。
  - 输入：
    - 关键词/品类
  - 输出：
    - 是否存在对应商品（有/无）
    - 价格区间、主流 SKU 数量
    - 简要机会位判断（如“已卖爆/竞争激烈”“中度竞争”“基本空白/机会大”）

- **N4：1688 工厂与拿货价查询**
  - 目标：为核价提供供应链视角，找到核心工厂与起批价区间。
  - 输入：
    - 产品关键词
    - 可选的特征：材质、尺寸、用途、是否需认证等
  - 输出：
    - 若干代表性工厂列表（名称、链接）
    - 起批价区间（含货币单位）
    - 典型 MOQ、发货地、是否含税等简要信息

### 二、选品与差异化策略分析（N5–N7）

- **N5：“非红海标品”识别与趋势分析**
  - 目标：从 N1 提供的飙升榜中，识别不是完全同质化红海、又有真实需求和增长势头的细分类目。
  - 输入：
    - N1 输出的候选商品列表
    - 可选参数：排除类目列表（如医药、监管严格类）
  - 输出：
    - 标记每个商品是否“红海标品”或“细分机会品”
    - 趋势说明：飙升原因、是否短期事件驱动（节日、热点）还是长期需求

- **N6：选品差异化策略判定**
  - 目标：为每个候选产品判定更适合采用哪种“避卷”策略。
  - 输入：
    - 候选产品基础信息（类目、价位段、是否标品、现有竞品形态）
    - N2/N3 输出的 Temu、4supply 竞品结构
  - 输出：
    - 策略类型：Bundle / Premium / Lightweight
    - 策略说明：建议组合哪些 SKU 做套装、可以升级哪些材质/外观、如何控制体积与重量

- **N7：安全合规与资质门槛识别**
  - 目标：提前过滤掉资质门槛过高、合规风险大的品类，提示必须的认证与注意事项。
  - 输入：
    - 产品类目、用途、目标市场
  - 输出：
    - 是否涉及高门槛资质（如医疗器械、婴幼儿、食品）
    - 需重点关注的认证（CE/FCC/FDA/ROHS 等）
    - 简要风险提示与避坑建议

### 三、V4.1 生存模型与核价（N8–N9）

- **N8：V4.1 倒推核价计算**
  - 目标：严格按题目给定公式，判定每个候选产品在 Temu 前台卷王价下是否达到净利红线。
  - 输入：
    - Temu 卷王价（单位：美元或人民币）
    - 1688 起批价（人民币）
    - 固定国内履约费：3.5 元
    - 汇率参数（如美元兑人民币，默认 7.2，可配置）
  - 核心公式：
    - 回款预估 = Temu 前台卷王价 × 45%
    - 若前台价为美元，则回款预估（人民币） ≈ 回款预估（美元）× 7.2
    - 总成本 = 1688 起批价 + 3.5
    - 净利润 = 回款预估（人民币） - 总成本
  - 输出：
    - 详细计算过程（含中间值）
    - 净利润数值
    - 是否满足净利 > 5 元的铁律（✅GO / ❌PASS）

- **N9：利润敏感性分析与调价建议（可选）**
  - 目标：帮运营理解定价对净利的敏感度，给出合理调价建议。
  - 输入：
    - 当前 Temu 卷王价
    - 1688 起批价
    - 可选的价格区间或多个备选定价
  - 输出：
    - 不同前台定价下的净利变化列表
    - 建议的目标定价/折扣范围

### 四、前端营销 / 说明文案生成（N10–N11）

- **N10：前端营销逻辑生成**
  - 目标：按题目要求的“第一部分：前端营销逻辑”结构，对每个候选产品生成给用户/运营看的卖点说明。
  - 输入：
    - 产品基础信息（来自 N1/N2/N3/N6/N7）
    - 目标人群与场景（如家庭办公、学生宿舍）
  - 输出：
    - 亚马逊热度说明（飙升原因、适用场景、节日/趋势因素）
    - 蓝海竞争说明（与 Temu 现有低价货的差异点）
    - 安全合规要点提炼（调用 N7 结果）
    - 给采购的“义乌/工厂找货指令”（材质、功能、避坑点）

- **N11：后端生存逻辑与核价结果说明**
  - 目标：人类友好地解释 N8/N9 的核价结果，便于快速审核与决策。
  - 输入：
    - N8/N9 的计算结果
  - 输出：
    - Temu 卷王价、收入预估、成本拆解、净利
    - GO / PASS 结论与简要原因

### 五、Orchestrator 与工作流辅助（N12–N13）

- **N12：候选产品自动筛选与排序 orchestrator**
  - 目标：把 N1–N11 串起来，从“给出选品请求”到“产出 5 个完整推荐”。
  - 输入：
    - 目标站点与类目（如 Amazon US 家具类）
    - 需要返回的产品数量（默认 5）
    - 策略参数（是否启用 N9 敏感性分析等）
  - 流程概要：
    - Step1：调用 N1 获取飙升榜候选池；
    - Step2：调用 N5/N7 过滤红海、高资质门槛产品；
    - Step3：对剩余候选品调用 N2/N3/N4；
    - Step4：调用 N8（可选叠加 N9）做 V4.1 生存模型核价；
    - Step5：过滤掉未达标（❌PASS）的品；
    - Step6：对剩余品按综合得分排序，挑选前 5 个；
    - Step7：对每个入选品调用 N10/N11 生成完整两部分输出。

- **N13：人工 Review 辅助（可选）**
  - 目标：为人工运营或采购提供一个快速浏览视图，便于做最终是否上线决策。
  - 输入：
    - 已筛选排序后的候选产品列表（含关键指标）
  - 输出：
    - 每个产品的关键指标摘要（销量趋势、利润、安全合规风险等）
    - 适合“强推”“观察”“放弃”的标签建议
